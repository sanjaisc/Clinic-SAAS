   301‚Üí## Authentication & Authorization
   302‚Üí
   303‚Üí### How Auth Works
   304‚Üí- **NextAuth v4** with **JWT strategy** (no session DB table)
   305‚Üí- Custom `toInternalRequest()` adapter in `/api/auth/[...nextauth]/route.ts` for Next.js 16 compatibility (bypasses broken `req instanceof Request` check)
   306‚Üí- Credentials provider: email + bcrypt password verification
   307‚Üí- JWT payload includes: `userId`, `email`, `name`, `role`, `clinicId`
   308‚Üí- Session expiry: 30 days
   309‚Üí- Token revocation: middleware checks `user.isActive` via DB on protected routes
   310‚Üí
   311‚Üí### Role Hierarchy
   312‚Üí```
   313‚ÜíSYSTEM_MANAGER > CLINIC_ADMIN > CLINIC_RECEPTION
   314‚Üí```
   315‚Üí
   316‚Üí### Middleware Enforcement
   317‚ÜíEdge middleware (`src/middleware.ts`) validates JWT and enforces:
   318‚Üí- **Public routes**: `/`, `/book`, `/clinic/*`, `/providers/*`, `/clinics`, `/about`, `/insurance`, `/manage/*`, `/intake/*`, `/review/*`, `/api/search/*`, `/api/clinics`, `/api/taxonomies`, `/api/specialties/*`, `/api/slots/*`, `/api/appointments` (POST only), `/api/waitlist`, `/api/intake`, `/api/reviews`, `/api/providers/*/availability`, `/api/providers/*/reviews`, `/api/geocode`, `/api/auth/*`
   319‚Üí- **Staff routes**: `/staff/dashboard/*`, `/staff/login`, `/staff/accept-invitation`
   320‚Üí- **Admin-only routes**: `/staff/dashboard/admin/*`, `/api/staff/admin/*`
   321‚Üí- Clinic status check: redirects to `/staff/login` if clinic is not PUBLISHED
   322‚Üí
   323‚Üí### Known Auth Quirk
   324‚ÜíThe `output: "standalone"` build requires `next-auth` and `cookie` in `serverExternalPackages` (next.config.ts). The root layout uses a `client-providers.tsx` wrapper with `next/dynamic({ ssr: false })` to bypass React 19 static generation issues.

   749‚Üí## Booking Flow
   750‚Üí
   751‚Üí### Two-Phase Lock Protocol
   752‚Üí
   753‚Üí```
   754‚ÜíPatient clicks "Book" on a slot
   755‚Üí         ‚îÇ
   756‚Üí         ‚ñº
   757‚Üí  POST /api/slots/[slotId]/lock
   758‚Üí  ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
   759‚Üí  ‚îÇ 1. Check slot is AVAILABLE    ‚îÇ
   760‚Üí  ‚îÇ 2. Create SlotLock (unique)   ‚îÇ
   761‚Üí  ‚îÇ    - lockKey: random token     ‚îÇ
   762‚Üí  ‚îÇ    - expiresAt: now + TTL      ‚îÇ
   763‚Üí  ‚îÇ 3. Update slot ‚Üí LOCKED       ‚îÇ
   764‚Üí  ‚îÇ 4. Return lockKey to client    ‚îÇ
   765‚Üí  ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
   766‚Üí         ‚îÇ
   767‚Üí         ‚ñº
   768‚Üí  Patient fills booking form (3 steps)
   769‚Üí         ‚îÇ
   770‚Üí         ‚ñº
   771‚Üí  POST /api/appointments
   772‚Üí  ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
   773‚Üí  ‚îÇ 1. Verify lockKey matches     ‚îÇ
   774‚Üí  ‚îÇ 2. Check lock not expired     ‚îÇ
   775‚Üí  ‚îÇ 3. Verify slot is LOCKED      ‚îÇ
   776‚Üí  ‚îÇ 4. Verify service‚Üíprovider    ‚îÇ
   777‚Üí  ‚îÇ 5. Create Appointment         ‚îÇ
   778‚Üí  ‚îÇ 6. Create AppointmentLedger   ‚îÇ
   779‚Üí  ‚îÇ 7. Delete lock                ‚îÇ
   780‚Üí  ‚îÇ 8. Update slot ‚Üí BOOKED       ‚îÇ
   781‚Üí  ‚îÇ 9. Generate 4 tokens:         ‚îÇ
   782‚Üí  ‚îÇ    MANAGE, INTAKE, REVIEW,    ‚îÇ
   783‚Üí  ‚îÇ    CHECK_IN (SHA-256 hashed)  ‚îÇ
   784‚Üí  ‚îÇ 10. Audit log                 ‚îÇ
   785‚Üí  ‚îÇ 11. Invalidate caches         ‚îÇ
   786‚Üí  ‚îÇ 12. Return appointment + token‚îÇ
   787‚Üí  ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
   788‚Üí```
   789‚Üí
   790‚Üí### Race Condition Protection
   791‚Üí- `SlotLock.slotId` has a **unique constraint** ‚Äî only one lock per slot
   792‚Üí- DB-level uniqueness prevents double-locking under concurrent requests
   793‚Üí- Expired locks are swept by cron job (or manually via `/api/admin/locks/cleanup`)

   797‚Üí## Appointment State Machine
   798‚Üí
   799‚Üí```
   800‚Üí        ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
   801‚Üí        ‚îÇ  BOOKED  ‚îÇ
   802‚Üí        ‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚î¨‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
   803‚Üí             ‚îÇ
   804‚Üí    ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îº‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚î¨‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
   805‚Üí    ‚ñº        ‚ñº        ‚ñº          ‚ñº
   806‚ÜíCHECKED_IN CANCELLED  NO_SHOW  (time passes)
   807‚Üí    ‚îÇ        (end)    ‚îÇ
   808‚Üí    ‚ñº                 ‚ñº
   809‚ÜíCOMPLETED          ARCHIVED
   810‚Üí    ‚îÇ
   811‚Üí    ‚ñº
   812‚Üí ARCHIVED
   813‚Üí```
   814‚Üí
   815‚ÜíValid transitions (enforced by `canTransitionTo()` in `enums.ts`):
   816‚Üí- BOOKED ‚Üí CHECKED_IN, CANCELLED, NO_SHOW
   817‚Üí- CHECKED_IN ‚Üí COMPLETED
   818‚Üí- COMPLETED ‚Üí ARCHIVED
   819‚Üí- NO_SHOW ‚Üí ARCHIVED
   820‚Üí
   821‚Üí**Admin override**: `/api/staff/admin/appointments/[id]/status` bypasses the state machine.

   839‚Üí## Known Limitations & Quirks
   840‚Üí
   841‚Üí1. **Email is mock**: `sendStaffEmail()` logs to console. Replace with real email service for production.
   842‚Üí
   843‚Üí2. **No Stripe integration**: Payment fields exist in schema but no actual payment processing. `MANUAL_WAIVER` payment method is used for demo bookings.
   844‚Üí
   845‚Üí3. **Dev server OOM**: In the Z.ai sandbox, `next dev` (Turbopack) occasionally runs out of memory. Production builds (`next build` + `node .next/standalone/server.js`) are more stable.
   846‚Üí
   847‚Üí4. **React 19 + NextAuth v4**: The `/api/auth/[...nextauth]/route.ts` has a custom `toInternalRequest()` adapter because NextAuth v4's internal function checks `req instanceof Request`, which fails across module boundaries in Next.js 16 production builds (different `Request` constructors).
   848‚Üí
   849‚Üí5. **Standalone build + static gen**: React 19's static generation fails when the root layout wraps pages with client-side providers (SessionProvider/ThemeProvider). Solved by `client-providers.tsx` using `next/dynamic({ ssr: false })`.
   850‚Üí
   851‚Üí6. **SQLite limitations**: No native ENUM support (all enums are application-level string constants). No concurrent write support beyond SQLite's WAL mode defaults.
   852‚Üí
   853‚Üí7. **Geocoding rate limits**: Nominatim (OSM) has a 1 request/second rate limit. The search page caches geocode results.
   854‚Üí
   855‚Üí8. **Slot generation**: Slots are pre-generated from templates for 90 days. A cron job (`/api/admin/slots/generate`) creates slot instances. If templates change, regenerate is needed.
   856‚Üí
   857‚Üí9. **`typescript.ignoreBuildErrors: true`**: The build skips TypeScript checking. Run `npx tsc --noEmit` separately for type checking.
   858‚Üí
   859‚Üí10. **Middleware deprecation**: Next.js 16 shows a warning that `middleware.ts` should be migrated to the `proxy` convention. Not yet migrated.
